## 思路

### 方法一：Set

利用 Set 自动去重的特性，最简洁高效。

- 时间复杂度 $O(n)$
- 空间复杂度 $O(n)$

### 方法二：includes

遍历数组，使用 `includes` 判断元素是否已存在于结果数组中。

- 时间复杂度 $O(n^2)$（includes 本身是 $O(n)$）
- 空间复杂度 $O(n)$

### 方法三：filter + indexOf

使用 `filter` 过滤，保留首次出现的元素（`indexOf` 返回首次出现的索引）。

- 时间复杂度 $O(n^2)$
- 空间复杂度 $O(n)$

### 方法四：Map

使用 Map 存储元素，利用 Map 的 key 唯一性去重。

- 时间复杂度 $O(n)$
- 空间复杂度 $O(n)$

### 关键点

- Set 和 Map 方法性能最优，时间复杂度 $O(n)$
- includes 和 indexOf 方法需要嵌套遍历，时间复杂度 $O(n^2)$
- 实际开发中推荐使用 Set 方法
